- The current config (min_confluence=0.20, min_confirmations=2,

- **Live validation** is now the priority — we need to confirm

## Refinement ideas (to test, in order of expected impact)

1. **Tighter confluence threshold** (0.20 → 0.25): filter weak
   signals. Expected: fewer trades, slightly higher win rate.
2. **More confirmations** (2 → 3): require more subsystems to
   agree. Expected: significantly fewer trades, possibly higher
   win rate but lower absolute PnL.
3. **Regime-conditional threshold**: drop confluence to 0.20
   in trending regimes, raise to 0.30 in ranging/CRISIS.
4. **Asymmetric SL/TP** (currently 2%/4%): keep 2% SL, raise
   TP to 5% or 6% in trending regimes to capture more of the
   move (compensates for slightly lower win rate).
5. **Volume-weighted position sizing**: scale position inversely
   with signal confidence (less size on lower-conviction
   trades). Need to wire up.

## Decision
- **Do NOT over-optimize against the backtest** — 75% OOS wins
  with 8 windows is statistically thin. Each refinement must
  be validated against another out-of-sample window.
- **Run the live bot** to gather closed trades. Aim for 20+
  closed trades before making any parameter changes.
- **When 20+ closed trades are in**:
  - If win rate ≥70%: keep current config
  - If 50-70%: pick the most conservative refinement (#1 or #4)
  - If <50%: bigger structural change (regime filter, etc.)
